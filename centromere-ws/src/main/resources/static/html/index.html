Welcome to Centromere

Genomic Data Warehouse and RESTful API

Getting started

Centromere is up-and-running and ready to use. Follow the instructions below to take a tour of the web service and its features.

  • Querying the Centromere API
  • Filtering search results
  • Outputting XML and plain-text
  • Hypermedia and resource discovery
  • Method-based security

Querying the Centromere API

Each model class in Centromere is represented by a single web service endpoint. To query the Genes repository, simple make a request to the root resource URL:


curl "http://localhost:8080/api/genes?size=10"


You can filter, sort, and page results using query string parameters. Most model attributes are filterable, using the field name as a query string parameter:


curl http://localhost:8080/api/genes
   ?geneType=protein-coding
   &sort=entrezGeneId,asc
   &page=2&size=100

Filtering search results

If you do no want to all of the target model's attributes to be returned, you can selectively filter them out with the exclude query string parameter. If you want only to return specific attributes, you can use the fields parameter:


curl http://localhost:8080/api/genes
   ?fields=entrezGeneId,primaryGeneSymbol,aliases
   &size=10&sort=entrezGeneId,asc

Outputting XML and plain-text

If no media type is specified in the Accept header, then application/json will be used.Centromere also dataSetSupport XML and tab-delimited plain-text output using application/xml and text/plain respectively:


curl -H "Accept: text/plain"
   http://localhost:8080/api/mutations?size=100

Hypermedia and resource discovery

Centromere uses Spring HATEOAS to generate HAL-formatted links associated with each entity record and resource collection. To view model hypermedia, use the associated media types, application/hal+json and application/hal+xml:


curl -H "Accept: application/hal+json"
   http://localhost:8080/api/mutations?size=10

Method-based security

Centromere intercepts HTTP requests and will filter them based upon security configuration. By default, read-requests will all be accepted (GET, HEAD, and OPTIONS), while write-requests will be denied, ubless properly authenticated (POST, PUT, and DELETE). An authentication token can be acquired by posting user credentials to the /authenticate address:


curl -X POST -u username:password
   http://localhost:8080/authenticate


The generated token may then be used to make requests to secured endpoints:


curl -X POST
   -H "X-Auth-Token: username:24243563:grw86e7rg59ew5gb0q9gwg"
   -H "Content-Type: application/json"
   -d '{"name": "test"}'
   http://localhost:8080/api/subjects

Put Centromere to Work for Your Data

Centromere is modular and extensible. It is developed in Java using the enterprise-grade Spring Framework. Centromere is designed to make developing data warehouses and web services as easy as possible, while ensuring long-term scalability and dataSetSupport.

Create a warehouse for you genomic data from scratch using SQL or NoSQL technologies, or bootstrap your existing databases using Centromere's repository API.

Define your own data model and expose as much or as little of your data via customizable RESTful API. Automatic web service security and API documentation are available right out of the box.

Centromere is free and open source. Visit the GitHub repository to get the latests build, or contribute your own code.
